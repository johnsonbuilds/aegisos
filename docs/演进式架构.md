# AegisOS 演进式架构

## 第一阶段（现在：MVP 极速狂飙期）

- **核心语言**：Python 3.11+
- **包管理器**：强烈建议使用 **uv** 或 **Poetry**（避免使用原生的 pip 和 requirements.txt，以防依赖冲突）。
- **多 Agent 调度**：抛弃传统多线程，全面拥抱原生 **asyncio**。AegisOS 的总线必须是异步的（async/await），以高效处理 Agent 等待大模型返回及文件 I/O 等密集型任务。
- **数据校验**：使用 **Pydantic v2** 作为全系统基石，定义所有 AACP 协议和 Agent 状态。

## 第二阶段（未来：当 AegisOS 成为真正的基础设施）

当生态爆发，需要后台同时调度上百个 Agent 且要求极致安全沙箱时，将采用多语言微服务架构：

- **Control Plane（控制面板/消息总线）**：使用 **Go** 或 **Rust** 重写核心调度器 `AegisDispatcher`。使其成为占用内存极低（约 20MB）且极其稳定的系统守护进程。
- **Agent Plane（智能体逻辑层）**：所有的 Agent 技能（Skills）和大脑（LLM 交互层）保留 **Python** 开发，通过本地 Socket 或 gRPC 与总线通信。
