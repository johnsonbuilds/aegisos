**1. 演进式网关设计 (Evolutionary Egress Gateway)**

- **概念**：在 AegisDispatcher 中抽象出一层 Network Gateway，对上层 Agent 屏蔽底层的网络寻址细节。当接收方 URI 包含非本地的 instance_id 时，自动触发网关路由。
    
    阶段1：个人设备集群（“私有云”模式 —— 推荐早期 MVP）
    
    - **底层技术**：**Tailscale / WireGuard (虚拟局域网)**
    - **怎么路由**：你不需要自己写复杂的 P2P 逻辑。用户在自己的手机、Mac 和云服务器上安装 Tailscale。Tailscale 会给每个设备分配一个固定的虚拟 IP（比如 100.x.x.x）。
    - **AegisOS 的设计**：
        - instance_id 直接就是虚拟 IP 或 Tailscale 的机器名（如 planner@macbook-pro.local）。
        - 调度器直接通过普通的 TCP Socket 或 gRPC 连接这个虚拟 IP。
    - **优点**：代码极简，安全性极其无敌（原生端到端加密，黑客根本连不进你的虚拟网）。
    - **缺点**：不适合陌生人（比如你的 Agent 想和别家公司的 Agent 砍价），只能用于你自己的设备。
    
    阶段2：联邦中继总线（“Nostr / Matrix”模式 —— 推荐商业化）
    
    - **底层技术**：**WebSocket + Pub/Sub Relay (发布订阅中继站)**
    - **怎么路由**：我们不搞极端的纯 P2P，而是像著名的去中心化社交协议 Nostr 一样，部署几个开源的中继服务器（Relays，比如 wss://relay.aegisos.com）。
    - **AegisOS 的设计**：
        - 所有在线的 Aegis 实例都通过 WebSocket 连着某个 Relay。
        - instance_id 是一串公钥。
        - 当北京的 Agent 要给新加坡的 Agent 发消息时，它把消息用对方的公钥**加密**，然后扔给 Relay 服务器：“帮我广播给拥有这个公钥的实例”。新加坡的实例收到后用私钥解密。
    - **优点**：完美解决穿透问题，跨公司、跨防火墙协作毫无压力。因为数据是加密的，中继站也不知道你们的 Agent 在聊什么机密。
    
    阶段3：纯粹的 Web3 级点对点（“IPFS / Libp2p”模式 —— 终极愿景）
    
    - **底层技术**：[**Libp2p**](https://www.google.com/url?sa=E&q=https%3A%2F%2Flibp2p.io%2F) (以太坊和 IPFS 的底层网络协议，有成熟的 Python 库 py-libp2p)。
    - **怎么路由**：完全采用刚才提到的 DHT 算法和 WebRTC 打洞。
    - **AegisOS 的设计**：
        - 引入完整的 P2P 网络栈。每个 AegisOS 实例启动时，作为一个 P2P 节点加入全球 Aegis 网络。
        - 调度器的 send_to_remote 直接调用 libp2p.send(peer_id, message)。
    - **优点**：最纯粹的去中心化 Agent 操作系统，抗审查，永远不会因为官方服务器宕机而瘫痪。
    - **缺点**：引入了极高的系统复杂度和资源占用，对 Python 环境的挑战较大。

为了保证我们现在的代码以后不需要推翻重写，我们的设计应该**对上层逻辑屏蔽底层的网络细节**。

不管你未来用哪种方案，在当前的 Phase 1 和 Phase 2 中，我们只需在 AegisDispatcher 里做一层抽象的**“网络网关 (Egress Gateway)”**：

```jsx
# dispatcher.py 伪代码

async def send_to_remote(receiver_uri: str, message: AACPMessage):
    # 解析 URI: {role}_{uuid}@{instance_id}
    _, instance_id = receiver_uri.split("@")
    
    # 我们预留一个配置接口，未来可以无缝切换底层网络引擎
    if CONFIG.network_mode == "tailscale":
        await send_via_grpc(instance_id, message)
    elif CONFIG.network_mode == "nostr_relay":
        await send_via_websocket(instance_id, message)
    elif CONFIG.network_mode == "libp2p":
        await send_via_dht(instance_id, message)
```

**2. AI 逻辑 + 底层 P2P 网络混合架构 (Python + Go Sidecar)**

- **概念**：放弃用 Python 处理高并发 P2P 状态机。采用“边车模式 (Sidecar)”：主进程 (Python) 专注执行 AI 逻辑、Agent 调度与沙箱控制；独立的守护进程 (Go 语言基于 go-libp2p 实现) 专注处理 Kademlia DHT 寻址、NAT 打洞 (Hole Punching) 及节点保活。
- **更新目标**：作为重大决策补充到 decisions.md 中。

**3. 跨语言通信 (IPC via UDS/gRPC)**

- **概念**：Python 主进程与 Go 网络守护进程之间，不通过外部网络，而是通过极低延迟的本地 Unix Domain Socket (UDS) 或 localhost gRPC 进行跨进程通信 (IPC)。
    
    **具体架构流转图：**
    
    1. **Python 侧 (AegisOS 主进程)**：专注跑 AI、调度 Agent、执行沙箱。
    2. **Go 侧 (AegisOS-Net 守护进程)**：只负责一件事——维护 Libp2p 网络、DHT 寻址、打洞、保持 WebSocket 长连接。
    3. **沟通桥梁 (IPC)**：Python 的“网络网关 (Egress Gateway)”通过本地的 **Unix Domain Socket (UDS)** 或 **localhost gRPC** 与同一个机器上的 Go 进程通信。
    
    **代码级伪逻辑：**
    
    ```jsx
    # Python 侧: dispatcher.py
    async def send_to_remote_via_go(receiver_uri: str, message: AACPMessage):
        # Python 不管什么打洞和DHT，直接把 JSON 扔给本地的 Go 进程
        async with grpc.aio.insecure_channel('unix:///tmp/aegisos_net.sock') as channel:
            stub = NetGatewayStub(channel)
            await stub.RouteMessage(receiver_uri, message.model_dump_json())
    ```

**4. 预编译捆绑的一键安装体验 (Pre-compiled Binary Bundling)**

- **概念**：为了保持 Python 包的一键安装体验（uv add aegisos），CI/CD 会将 Go 网络引擎交叉编译为各平台（Linux, macOS ARM, Windows）的微型可执行文件，并直接打包进 Python 的 .whl 产物中（置于 src/aegisos/bin/）。Python 的 Launcher 启动时，会自动根据当前操作系统，通过 subprocess 在后台拉起对应的 Go 进程。
    
    **打包与部署流水线：**
    
    1. **GitHub Actions 自动化编译**：
        
        当你要发布新版本时，CI/CD 流水线会用 Go 将网络层代码交叉编译成各平台的独立可执行文件：
        
        - aegisos-net-linux-amd64
        - aegisos-net-darwin-arm64 (Mac M1/M2)
        - aegisos-net-windows.exe
    2. **打入 Python Wheel 包**：
        
        在打包 Python 代码（构建 .whl 文件）时，把这些体积很小（Go 编译出来通常只有 10-20MB）的二进制文件直接塞进 src/aegisos/bin/ 目录下。
        
    3. **用户端的无感启动 (Subprocess 托管)**：
        
        当用户在终端输入 aegisos start 启动系统时，Python 里的代码会自动识别当前操作系统，找到对应的 Go 二进制文件，并在后台悄悄启动它。
        
    
    **启动代码示例：**
    
    ```jsx
    # src/aegisos/core/launcher.py
    import subprocess
    import sys
    import platform
    from pathlib import Path
    
    def start_go_network_daemon():
        # 自动定位打在 pip 包里的 Go 二进制程序
        bin_dir = Path(__file__).parent.parent / "bin"
        
        if platform.system() == "Darwin" and platform.machine() == "arm64":
            executable = bin_dir / "aegisos-net-darwin-arm64"
        elif platform.system() == "Windows":
            executable = bin_dir / "aegisos-net-windows.exe"
        # ... 其他系统判定
        
        # 以后台子进程的形式拉起 Go 网络引擎
        # Go 进程的生命周期与 Python 主进程绑定（主进程死，Go 进程自动被回收）
        return subprocess.Popen([executable, "--socket", "/tmp/aegisos_net.sock"])
    ```
